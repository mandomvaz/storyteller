## Why

Stories are generated by the LLM pipeline but never persisted. The user wants to keep their generated stories, view a list of past stories, and retrieve individual stories. The app runs on a Docker-based minipc, so everything must be local, lightweight, and configurable via volumes.

## What Changes

- **Enhanced `Story` model**: Add `Id` (Guid), `Badge` (3 emojis via SK), `CreatedAt`. `Title` and `Paragraphs` already exist.
- **SQLite database**: Raw `Microsoft.Data.Sqlite` for persistence. Configurable path via `appsettings.json` + env var override. One table with Paragraphs as JSON.
- **`IStoryRepository` / `SqliteStoryRepository`**: Save, list (id + badge), get by id.
- **`BadgeService`**: Calls SK via injected `KernelFunction` to generate 3 emojis summarizing the story. Prompt is baked into the function at startup (from a configurable file path).
- **`PersistenceService`**: Two-phase save: (1) cache story in `IMemoryCache` (10min TTL) right after badge generation; (2) on user confirmation, persist from cache to SQLite.
- **`POST /api/stories/{id}/save`**: New endpoint that triggers persistence from cache to DB.
- **Config paths**: DB path and badge prompt file path configurable in `appsettings.json`, overridable via env vars for Docker.

## Capabilities

### New Capabilities
- `story-model`: The Story entity — fields, serialization, and lifecycle
- `story-repository`: SQLite repository for stories (save, list summaries, get by id)
- `badge-generation`: SK-based badge generation — 3 emojis from story title + full text, prompt read from file on startup
- `story-persistence-service`: MemoryCache-backed deferred persistence with 10min TTL
- `story-save-api`: HTTP endpoint to confirm story save

### Modified Capabilities
- `story-generation`: The existing spec covers story generation. The `Story` model is enriched with `Id`, `Badge`, `CreatedAt`, and a badge generation step is added after story text is complete. The pipeline runner captures the returned `Story` and passes it to the badge/persist services.

## Impact

- **New NuGet dependency**: `Microsoft.Data.Sqlite`
- **New services**: `IStoryRepository`, `SqliteStoryRepository`, `BadgeService`, `PersistenceService`
- **Updated model**: `Story.cs` gains `Id`, `Badge`, `CreatedAt`, `Paragraphs` serialization
- **Updated pipeline runner**: After `storyTask` completes, calls BadgeService then PersistenceService
- **New config sections**: `Database:Path`, `BadgePrompt:Path`
- **New API route**: `POST /api/stories/{id}/save`
- **Docker**: Volumes for `/app/Data` (SQLite) and `/app/Prompts` (badge prompt)
